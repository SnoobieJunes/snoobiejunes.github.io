🎴 The Gallows Way App Clip Flow

From challenge link to full game installation

INVITER (Full App)
1

Generate Challenge Link

Player in the full app taps "Online" → "Challenge a Friend" → generates a shareable link with their Game Center ID.
https://snoobiejunes.github.io/challenge?from=GPID&name=Alice
✓ Link contains inviter's own gamePlayerID, not a new ID
→
RECIPIENT (App Clip)
2

Tap Challenge Link

Friend receives the universal link and taps it. iOS launches the App Clip with the URL parsed into a ClipChallenge object.
handleChallengeURL(url) → ClipChallenge( inviterGamePlayerID: "...", inviterName: "Alice" )
✓ App Clip launches with no Game Center auth yet
→
RECIPIENT (App Clip)
3

ClipWelcomeView

Shows challenger's name ("Alice challenged you to a game") and an "Accept Challenge" button. A 30-second watchdog guards against iOS 26 auth regression.
User taps "Sign In & Play" → GameKitManager.authenticate()
✓ 30s watchdog prevents auth handler timeout
→
RECIPIENT (App Clip)
4

Game Center Auth

User signs in with Apple ID. isAuthenticated flips to true, which triggers the accept logic.
⚠️ CRITICAL: User must be signed into Game Center. No custom account needed.
→
RECIPIENT (App Clip)
5

One-Match-Per-Pair Check

Before creating a new match, check if a match already exists with this inviter. If found, adopt it instead (prevents duplicates).
adoptExistingMatch( withGamePlayerID: inviter_gpid )
✓ Tapping the same link twice resumes, not creates duplicate
Match exists? → YES: Skip to Step 9 (resume)
Match exists? → NO: Continue to Step 6 (create new)
→
RECIPIENT (App Clip)
6

Create Challenge Match

Create a new GKTurnBasedMatch with minPlayers=2, maxPlayers=2 (strictly 1v1). The recipient is automatically the currentParticipant (host).
createChallengeMatch( invitingGamePlayerID: gpid ) → GKTurnBasedMatch created → Recipient is HOST
✓ Recipient becomes the HOST, not the guest
→
GAME START
7

Host Deals Immediately

No pre-game lobby or "ready" screen. The host (recipient) immediately builds GameState from match.participants and deals cards.
state.startLocalGame() → Deck shuffled → Cards dealt
✓ This is the critical GameKit pattern: deal first, no lobby
→
GAME START
8

Host Ends Turn to Inviter

Host sends their game snapshot and ends their turn, making the inviter the currentParticipant. This is what delivers the push notification.
gameKitManager.send(snapshot) → match.endTurn( withNextParticipants: [inviter] )
🎯 GAMEKIT CONSTRAINT (bug #27):
Invitations are ONLY delivered when the invited player becomes currentParticipant.
✓ Inviter's device receives push notification now
→
INVITER (Full App)
9

Push Notification Arrives

Inviter's device receives the "It's your turn in The Gallows Way" push notification. Tapping it opens the full app with the match loaded.
✓ Inviter joins the match as guest
→
GAME IN PROGRESS
10

Turn-Based Gameplay

Both players take turns. The host's snapshot is the single source of truth. Each endTurn rotates to the next player's device. Uses the shared GameView and full game engine.
✓ Clip user experiences the REAL game, not a demo
✓ Game recorded in Game Center history
→
INSTALL FUNNEL
11

Game Reaches Conclusion

When gameState.phase == .gameOver, the ClipRouterView swaps from GameView to ClipInstallFunnelView.
if phase == .gameOver → ClipInstallFunnelView
→
INSTALL FUNNEL
12

Result Screen + SKOverlay

Shows the final score and result (win/loss), then presents Apple's SKOverlay install card. The message explains the match is saved in Game Center history.
✓ Match persists even if user doesn't install
✓ If user installs later with same Apple ID, match appears in history
→
INSTALL FUNNEL
13

User Chooses

User either installs the full app (taps "Get The Gallows Way") or declines (taps "No thanks" → "Thanks for playing" message).
Install app? → YES: Full app opens, user continues
Install app? → NO: Thanks message + option to install later
Legend
Inviter actions (Full app)
Setup phase (App Clip auth & match creation)
Game in progress (Shared game engine)
Install funnel (Result → SKOverlay)
🔑 Key Insights

1. Recipient becomes host: The person tapping the link is NOT the guest—they're the host who deals first. This flips the typical invite pattern.

2. Push delivery is the bottleneck: GameKit only sends invites when the invitee becomes currentParticipant. The host's endTurn is what triggers the notification.

3. No pre-game lobby: There's no "wait for everyone to be ready." The host deals immediately. This is a feature, not a limitation—it respects the GameKit constraint.

4. Real game in 15MB: The clip runs the full game engine, not a trivial demo. Users get to experience the real card game mechanics.

5. Persistence across installs: Because it's a real GKTurnBasedMatch, the game is recorded in Game Center. If the user installs the full app later with the same Apple ID, the match is waiting for them.
